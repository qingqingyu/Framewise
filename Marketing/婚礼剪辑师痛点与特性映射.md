## 一、六大脏活累活

### 1. 素材整理与分类

**痛点描述**：面对 8 小时、4 台摄像机的素材，根本不知道从哪里下手。

> "我有大型婚礼要在一月底交稿，光是把 8 小时 4 机位的素材铺到时间线上，就已经让我陷入剪辑瘫痪（editing paralysis）了。"

- 把素材手动归类进各个 bin（新娘准备、新郎准备、仪式、晚宴等）
- 看完所有素材才知道手里有什么
- 不同机位、不同摄像机的文件混在一起

### 2. 音频处理

**痛点描述**：真正的体力活，被多个帖子反复提及。

- **演讲和誓词的清理**：逐字逐句剪掉"嗯…"、停顿、口误，把磕磕巴巴的致辞打磨成流畅音频——极其耗时
- **多轨音频同步**：录音机、麦克风、摄像机音频需要手动对齐
- **音频漂移修正**：长时间拍摄中各设备时钟不同步导致的偏移
- **现场收音质量差**：收音距离远、环境噪音、麦克风摆放不当

### 3. 素材筛选（粗剪前的挑选）

**痛点描述**：介于"整理"和"剪辑"之间，极其耗时。

> "光是认真地以 2 倍速看完所有素材，就已经是一项巨大工程了。"

- 需要看完所有素材才敢动刀
- 要判断哪些镜头"没就位"（摄像师还没站好就开录）
- 无人机镜头的起飞阶段、抖动阶段需要裁掉
- 地面镜头片尾有"走路时不小心按录制"拍到地面的废料

### 4. 多机位同步与 Multicam 搭建

**痛点描述**：技术门槛高且耗时的机械工作。

- 不同摄像机时间码不一致
- 多台设备的音视频手动对齐
- 搭建干净的多机位时间线才能开始真正剪辑

### 5. 故事结构搭建（初版时间线）

**痛点描述**：面对空白时间线不知从何下手。

> "我需要先把音频要点（演讲高潮、誓词情绪点）选出来，然后围绕这些点来搭整个片子的骨架——这是最耗时间的环节。"

- 如何把一天的素材组织成有逻辑、有情感起伏的叙事
- 精华片段、全天纪录片、仪式、晚宴各需要单独的时间线

### 6. 重复性格式化工作

**痛点描述**：不起眼但反复出现的细碎工作。

- 导出适配不同平台的时间线格式
- 颜色空间不对需要手动重新套
- 文件命名、文件夹整理、备份

## 二、痛点 vs Framwise 当前能力

| 脏活累活 | Framwise 覆盖情况 | 对应功能 |
|---|---|---|
| **素材整理与分类** | 🟡 部分覆盖 | 自动场景切割 + 按源文件分组浏览 |
| **音频处理** | ⬜ 不覆盖 | 超出产品边界 |
| **素材筛选（粗剪前的挑选）** | 🟢 核心覆盖 | 网格浏览 + 缩略图轮播 + 选择 + 预览 |
| **多机位同步与 Multicam** | ⬜ 不覆盖 | 超出当前范围 |
| **故事结构搭建** | 🟡 部分覆盖 | 拖拽重排 + 导出 EDL/FCPXML |
| **重复性格式化工作** | 🟢 核心覆盖 | EDL + FCPXML 导出，智能文件命名 |

### Framwise 的核心价值定位

Framwise 完美命中了 **痛点 #3（素材筛选）**，这是婚礼剪辑师最耗时的纯体力环节：

```
传统流程：双击视频 → 看 → 标记 → 下一个视频 → 看 → 标记 → …（几小时）
Framwise：导入 → 自动切割成网格 → 扫一眼 → 点选 → 导出（几分钟）
```

## 四、特性缺口分析与下一步建议

从用户痛点中，可以提取出以下**高价值、可落地**的增强特性：

### 优先级 P0：废料检测（自动化排除无用片段）

**来源痛点**：素材筛选 #3
**用户原声**：
- "要判断哪些镜头'没就位'（摄像师还没站好就开录）"
- "无人机镜头的起飞阶段、抖动阶段需要裁掉"
- "地面镜头片尾有'走路时不小心按录制'拍到地面的废料"

**实现思路**：
- 检测画面模糊度（Laplacian 方差），自动标记模糊片段
- 检测画面亮度突变（镜头盖/地面），标记纯黑/纯色片段
- 检测画面运动幅度（光流），标记静止废料（摄像机放在桌上误录）
- 被标记的片段默认折叠或半透明，用户可一键展开查看

**价值**：8 小时素材中可能有 1-2 小时是废料，自动排除后网格更干净，选片更快。

### 优先级 P1：片段时长筛选

**来源痛点**：素材整理 #1 + 素材筛选 #3
**场景**：用户只想看 5-30 秒的"有用镜头"，过滤掉 0.5 秒的闪切和 3 分钟的固定长镜头

**实现思路**：
- 网格工具栏增加时长范围滑块（最小 ~ 最大秒数）
- 实时过滤显示符合时长范围的片段

**价值**：减少视觉噪音，聚焦有价值的镜头。

### 优先级 P2：导出前片段合并预览

**来源痛点**：故事结构搭建 #5
**场景**：选完片段后，想快速看一遍选出来的镜头连起来是什么效果

**实现思路**：
- 导出弹窗中增加"预览串联播放"按钮
- 按选中顺序（或拖拽排序后的顺序）连续播放所有已选片段
- 用户可以在预览中调整顺序后再导出

**价值**：减少"导出 → 导入 Premiere → 发现不对 → 回来改"的反复。

### 优先级 P3：片段标注/打标

**来源痛点**：素材整理 #1
**场景**：给片段打上"新娘准备"、"仪式"、"晚宴"、"第一支舞"等标签，方便分类查看

**实现思路**：
- 右键菜单增加"标记为…"选项
- 支持颜色标记 + 文字标签
- 网格支持按标签筛选/分组

**价值**：替代手动建 bin 的工作流，直接在 Framwise 中完成分类。

## 五、与 Quartz Studio 的差异化

| 能力 | Framwise | Quartz Studio |
|---|---|---|
| 素材自动分类 | 按场景切割 + 按源文件分组 | 按婚礼阶段自动分类（准备/仪式/舞蹈等） |
| 多机位同步 | 不覆盖 | 自动同步 + 漂移修正 |
| 无人机/未就位检测 | 不覆盖（P0 规划中） | 已做 |
| 音频转录辅助分类 | 不覆盖 | 已做 |
| 初版时间线搭建 | 拖拽排序 + 导出 | 自动生成（精华/全天/仪式） |
| 演讲音频清理 | 不覆盖 | 未提及 |
| 音乐节奏匹配剪切点 | 不覆盖 | 开发中 |
| 导出至剪辑软件 | EDL + FCPXML | 规划中 |
| 产品形态 | Mac 桌面 App，本地运行 | 不明确 |
| 上手门槛 | 拖入即用，无配置 | 不明确 |

**Framwise 的核心差异**：轻量、本地、开箱即用。不做全流程自动化，而是**把最耗时的体力环节（看素材 + 选片段）做到极致**，然后通过标准格式无缝对接用户已有的剪辑工作流。

## 六、总结

Framwise 目前精准命中了婚礼剪辑师最痛的一个环节——**素材筛选**。这是连接"拍摄完"和"开始剪辑"之间的桥梁，也是一个之前没有好工具的空白地带。

下一步应该：
1. **先上线 P0（废料检测）**，这是用户感知最强的"省时间"功能
2. **打磨核心体验**，确保"导入 → 浏览 → 选择 → 导出"这个流程足够丝滑
3. **用婚礼剪辑师群体验证**，确认产品方向后再扩展到其他用户群体
