# Framwise 三问裁剪法：产品边界拆解

> 每个特性用三个维度定义边界：**做什么**、**不做什么**、**做到什么程度**
> 目的：让每个功能的范围清晰、可交付、可验证
> 整理日期：2026-04-12

---

## 产品总边界

```
做什么：帮剪辑师快速浏览和筛选视频素材，导出到剪辑软件
不做什么：不做剪辑、不做音频处理、不做云端、不做AI成片
做到什么程度：覆盖"拍摄完 → 开始剪辑"之间的空白环节，做到极致丝滑
```

## 特性 1：素材导入

### 做什么
- 拖拽文件到窗口，或点击按钮选择文件
- 支持多文件同时导入
- 支持 MP4、MOV、MXF、AVI、MKV
- 导入后自动启动场景分析
- 实时显示导入进度和分析进度

### 不做什么
- 不做云端导入（不支持 URL 导入、不支持从云盘拉取）
- 不做批量文件夹扫描（用户必须主动选择/拖入文件）
- 不做导入前的格式转换（不支持的视频格式直接跳过，不提示转码）
- 不做项目持久化（关闭 App 后导入会话丢失，下次需重新导入）

### 做到什么程度
- **够了**：用户 3 秒内完成导入操作，进度条清晰可见
- **够了**：导入 1 小时视频，10 秒内开始看到第一批片段出现在网格中（流式分析）
- **不够**：如果导入失败没有友好提示 → 需要补错误提示
- **不够**：如果关闭 App 后重开要重新分析 → 可考虑缓存分析结果

## 特性 2：自动场景切割

### 做什么
- 用帧差分直方图比较法检测镜头切换点（cut点）
- 超过 5 秒的长镜头，每 5 秒额外切一刀
- 灵敏度可调（0.1~0.9，默认 0.3）
- 目标片段数可调（12~120，默认 36）
- 分析完成后自动展示网格

### 不做什么
- 不做语义级场景识别（不能区分"婚礼仪式"和"婚礼晚宴"）
- 不做人脸/物体检测（不能识别"新娘出现了"）
- 不做音频场景检测（不能根据音乐变化切分）
- 不做渐变转场检测（只检测硬切，不检测溶解/叠化）
- 不让用户手动添加/删除切点

### 做到什么程度
- **够了**：硬切场景检测准确率 >90%，大部分素材不需要用户干预
- **够了**：5 秒等分保证网格里每格时长均匀，方便浏览
- **不够**：渐变转场会漏检 → 对婚礼视频中的叠化转场可能不准确
- **够了**：灵敏度和片段数两个参数给用户足够的调整空间
- **判断**：当前程度已满足 v1 交付标准

## 特性 3：网格浏览

### 做什么
- 缩略图网格展示所有片段
- 每个片段自动轮播多张缩略图（2~6张，按时长自适应）
- 三档网格大小（小/中/大），列数自适应窗口宽度
- 按源文件分组显示，每组有标题栏
- 搜索（按文件名）、按源文件过滤
- 视图模式切换（全部 / 已选）
- 排序（原始顺序 / 时长 / 文件名）
- 迷你时间线导航（按源文件着色，点击定位）
- 懒加载，只渲染视野内的格子

### 不做什么
- 不做无限滚动加载历史项目（只有当前会话的片段）
- 不做拖拽调整列宽（只有预设三档）
- 不做列表视图/详情视图切换（只有网格）
- 不做自定义布局保存（窗口大小/网格大小不跨会话记忆）
- 不做片段标签/颜色分类

### 做到什么程度
- **够了**：100 个片段的网格滚动流畅（LazyVGrid + 懒加载）
- **够了**：搜索和过滤实时响应，无延迟
- **够了**：时间线导航让用户快速定位到时间区间
- **不够**：500+ 片段时的性能未验证 → 需要压测
- **判断**：当前程度已满足 v1 交付标准

## 特性 4：片段拖拽重排

### 做什么
- 拖拽片段到新位置，自由调整顺序
- 首次拖拽后自动从分组视图切换为扁平网格
- 视觉反馈：源片段半透明，目标片段蓝色边框
- 提供「恢复原始排序」按钮回到分组视图

### 不做什么
- 不做跨组拖拽的智能分组（拖拽不会自动归类）
- 不做撤销/重做（拖错了只能再拖回来或点恢复）
- 不做拖拽排序的持久化（关闭 App 后排序丢失）
- 不做片段间的吸附/对齐

### 做到什么程度
- **够了**：单次拖拽操作即时生效，无延迟
- **够了**：恢复按钮一键回到原始状态
- **不够**：没有撤销 → 误操作后只能手动恢复或全部重置
- **判断**：当前程度满足 v1，撤销功能可作为 v2 增强

## 特性 5：镜头选择

### 做什么
- 单击选中/取消
- 多选（无上限）
- 全选 / 取消全选 / 反选 / 按同文件全选
- 选中片段高亮显示
- 右键菜单提供快捷操作

### 不做什么
- 不做框选/区域选择（不能拖一个矩形框选多个）
- 不做按时间范围选择（不能选"10:00-20:00 之间的所有片段"）
- 不做选择集保存/命名（不能保存"第一版选择"然后另存一版）
- 不做选择状态的持久化

### 做到什么程度
- **够了**：单击 + 批量操作覆盖 95% 的选择场景
- **不够**：没有框选 → 大量选择时仍需逐个点击
- **不够**：没有选择集保存 → 反复调整时容易丢失之前的选择
- **判断**：当前程度满足 v1，框选和选择集保存是 v2 候选

## 特性 6：片段预览

### 做什么
- Space 键或右键打开预览弹窗
- 弹窗内嵌视频播放器，自动播放该片段时间范围
- 播放/暂停、进度条拖拽定位、回到起点
- 显示文件名和时间码

### 不做什么
- 不做全屏预览
- 不做画中画预览（不能边浏览网格边预览）
- 不做预览中截帧/打点
- 不做变速播放（不支持 2x/0.5x）
- 不做音频波形显示

### 做到什么程度
- **够了**：快速确认片段内容是否想要，3 秒内做出判断
- **不够**：只有弹窗模式 → 看预览时网格被遮挡，不能边看边选
- **不够**：没有变速 → 想快速扫完一个长片段时体验不好
- **判断**：当前程度满足 v1 的基本需求，画中画是 v2 高优

## 特性 7：导出

### 做什么
- 导出 EDL (CMX 3600)：兼容 Premiere / DaVinci / FCP
- 导出 FCPXML 1.9：兼容 DaVinci / FCP
- 智能文件命名（单源文件用源名，多源用时间戳）
- 记录原始文件名、入点出点时间码
- 支持多源文件混合导出

### 不做什么
- 不做 AAF 格式导出
- 不做 Premiere 原生项目文件导出
- 不做导出视频文件（不生成 MP4/MOV）
- 不做导出前的片段微调（不能在导出弹窗里微调入出点）
- 不做批量导出多个 EDL（一次只能导出一个文件）

### 做到什么程度
- **够了**：导出的 EDL/FCPXML 能被 Premiere 和 DaVinci 正确导入
- **够了**：时间码精度到帧级（CMTime 精度）
- **不够**：导出后用户无法在 Framwise 内验证正确性 → 需要信任导出结果
- **判断**：当前程度满足 v1，导出验证（预览串联播放）是 v2 候选

## 特性 8：缩略图系统（技术层）

### 做什么
- 每个片段生成 2~6 张缩略图，均匀分布
- 双层缓存：内存（LRU，500张）+ 磁盘（PNG，2GB 上限）
- 进入网格时批量预加载
- 自动裁剪/缩放到目标尺寸

### 不做什么
- 不做 GIF 动画导出
- 不做自定义缩略图时间点（用户不能手动选择哪几帧作为缩略图）
- 不做缩略图质量可调
- 不做缩略图缓存的手动清理入口

### 做到什么程度
- **够了**：网格滚动时缩略图即时显示，无白屏闪烁
- **够了**：磁盘缓存保证二次打开时缩略图秒出
- **判断**：当前程度已满足 v1

---

## 特性 9：时间码处理（技术层）

### 做什么
- 自动检测视频帧率
- 标准 HH:MM:SS:FF 格式输出
- 支持丢帧时间码（30fps drop-frame）
- EDL 用 CMX 3600 标准时间码
- FCPXML 用秒数格式

### 不做什么
- 不让用户手动指定帧率（自动检测）
- 不做自定义时间码起始偏移
- 不做时间码格式偏好设置

### 做到什么程度
- **够了**：主流帧率（24/25/30/60）正确处理
- **判断**：当前程度满足 v1

## 特性 10：废料检测

### 做什么
- 自动检测四类废料片段：黑屏（Blackout）、极暗（Dark）、纯色/地面（Solid）、模糊/失焦（Blurry）
- 在片段的 25%、50%、75% 三个时间点采样帧，计算平均亮度、亮度标准差、拉普拉斯方差
- 模糊检测算法：对 50×50 缩略图计算 Laplacian 方差（边缘响应强度），方差低于阈值判定为失焦
- 多数投票机制：3 帧中 ≥2 帧一致判定才标记为废料
- 分类优先级：黑屏 > 极暗 > 纯色 > 模糊（避免黑帧被误分类为模糊）
- 检测结果标记在 VideoClip 模型上（`wasteType` 字段）
- 在网格中以视觉方式区分废料片段

### 不做什么
- 不做运动检测/静态画面检测（camera-left-recording 场景未覆盖）
- 不做音频质量检测（不能识别"这段录音有噪音"）
- 不做自动删除/隐藏废料片段（只标记，不替用户做决定）
- 不做用户自定义废料阈值（黑屏 <8/255、极暗 <25/255、纯色 stdDev <5、模糊 Laplacian <100 不可调）
- 不做废料检测的开关控制（每次导入自动检测，不可关闭）

### 做到什么程度
- **够了**：黑屏和纯色画面检测准确率极高，误报率低
- **够了**：投票机制（≥2/3）避免单帧异常导致的误判
- **够了**：采样 50×50 像素缩略图做亮度+边缘分析，性能开销极小
- **够了**：模糊检测使用 Laplacian 方差，业界标准算法，对明显失焦镜头检测可靠
- **不够**：极暗片段和正常夜景之间可能误判 → 婚礼暗场仪式可能被误标
- **不够**：运动模糊（motion blur）和景深浅（bokeh）可能被误判为失焦 → 需要实际素材调阈值
- **不够**：缺少检测开关 → 用户无法跳过废料检测以加速导入流程
- **不够**：没有"标记为非废料"的手动纠正入口
- **判断**：模糊检测已实现，手动纠正和检测开关是下一步高优

## 汇总：边界一览

| 特性 | 做了 | 没做但重要 | v2 优先级 |
|---|---|---|---|
| 素材导入 | 多文件、拖拽、进度 | 错误提示、分析结果缓存 | P2 |
| 场景切割 | 直方图差分、灵敏/数量可调 | 渐变转场检测、手动调切点 | P3 |
| 网格浏览 | 分组/搜索/过滤/时间线 | 500+ 片段压测、框选 | P2 |
| 拖拽重排 | 自由拖拽、恢复排序 | 撤销/重做 | P3 |
| 镜头选择 | 批量选择、右键菜单 | 框选、选择集保存 | P2 |
| 片段预览 | 弹窗播放、进度拖拽 | 画中画、变速播放 | P1 |
| 导出 | EDL + FCPXML、智能命名 | 导出前串联预览、AAF 格式 | P1 |
| 缩略图 | 双层缓存、批量预加载 | 手动清理入口 | P3 |
| 时间码 | 多帧率、丢帧支持 | — | 已完成 |
| 废料检测 | 黑屏/极暗/纯色/模糊自动标记 | 手动纠正、检测开关 | P1 |

### v2 特性优先级排序

1. **P1 — 预览体验**：画中画预览 + ~~变速播放~~ ✅ + 导出前串联预览
2. **P1 — 废料检测增强**：~~模糊检测~~ ✅ + 手动纠正 + 检测开关 + 暗场防误判
3. **P2 — 选择增强**：框选 + 选择集保存 + 时长范围筛选
4. **P2 — 稳定性**：错误提示 + 大量片段压测 + 分析结果缓存
5. **P3 — 切割增强**：渐变转场 + 手动调切点 + 撤销重做
