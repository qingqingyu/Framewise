# Framwise 产品需求文档 v3

## 产品定位

一个Mac原生桌面App（Swift + SwiftUI），面向画面感剪辑师，帮助他们快速浏览和筛选本地视频素材，然后导出到PR/达芬奇进行精剪。

## 核心工作流

```
导入本地素材 → 自动切割成镜头网格 → 浏览筛选镜头 → 导出EDL/XML → 进PR/达芬奇精剪
```

## 功能需求

### 1. 素材导入
- 支持拖拽或选择本地视频文件导入（支持多文件同时导入）
- 素材文件始终保留在用户本地，不上传服务器
- 支持格式：MP4、MOV、MXF、AVI、MKV
- 导入过程实时显示进度（导入进度 + 分析进度）
- 导入完成后自动启动场景分析，实时显示分析状态

### 2. 自动切割逻辑（两层叠加）
- **第一层**：按镜头cut点自动切割，每个自然镜头为一格
- **第二层**：超过5秒的长镜头，每5秒再切一格
- 目的是保证网格里每一格时长大致均匀
- **场景检测灵敏度可调**（0.1~0.9，默认0.3）：灵敏度越高，检测到的切点越多
- **目标片段数可调**（12~120，默认36）：控制最终生成的片段数量

### 3. 网格浏览界面
- 所有镜头以缩略图形式展示在网格中
- 每个片段展示多张缩略图并自动轮播
- 只渲染用户视野内的格子（懒加载），保证流畅度
- **网格大小可调**：小 / 中 / 大 三档，列数自适应窗口宽度
- **按源文件分组**：多文件导入时，网格按源文件自动分组显示，每组显示文件名和片段数量
- **搜索功能**：按文件名搜索片段
- **按源文件过滤**：点击侧边栏可按源文件筛选网格中的片段
- **视图模式切换**：可切换查看「全部片段」或「已选片段」
- **排序方式**：支持按原始顺序 / 按时长 / 按文件名排序
- **迷你时间线导航**：网格顶部显示时间线条，按源文件着色，可点击快速定位到对应片段
- **键盘快捷键**：
  - `Space` 预览当前悬停片段
  - `Cmd+I` 打开导入
  - `Cmd+E` 打开导出

### 4. 片段拖拽重排
- 用户可拖拽片段自由调整排列顺序
- **首次拖拽后切换为扁平网格**：从默认的按源文件分组视图，切换为单层扁平网格，用户自定义排序
- 拖拽视觉反馈：拖拽中源片段半透明，目标片段蓝色边框高亮
- **恢复原始排序**：工具栏提供「恢复原始排序」按钮，清除自定义顺序回到按源文件分组视图

### 5. 镜头选择
- 单击选中/取消某一格镜头
- 支持多选
- 选中的镜头高亮显示
- **批量选择操作**：
  - 全选
  - 取消全选
  - 反选（基于当前显示的片段）
  - 按同文件全选（右键菜单）
- **右键菜单**：选中/取消、按同文件全选、预览

### 6. 片段预览
- 点击 Space 键或右键菜单可预览片段
- 预览弹窗内嵌视频播放器，自动播放该片段对应的时间范围
- 播放控制：播放/暂停、进度条拖拽定位、回到起点
- 显示片段文件名和时间码范围

### 7. 导出
- 用户选好镜头后，导出EDL或FCPXML文件
- 文件内记录原始素材文件名、每段片段的入点和出点时间码
- 导出的文件可直接导入Premiere Pro和DaVinci Resolve
- **智能命名**：当所有选中片段来自同一个源文件时，自动以源文件名作为导出文件名；多源文件时使用时间戳命名

## 技术要求

- **平台**：macOS原生，Swift + SwiftUI
- **视频处理**：AVFoundation框架（帧提取、缩略图生成）
- **渲染**：利用Metal保证网格滚动流畅
- **无需服务器**，纯本地运行

## 不做的事情（明确边界）

- 不做废料自动过滤（第二阶段再做）
- 不做时间轴剪辑功能
- 不做云端存储
- 不做自动成片
- 不做口播/字幕相关功能

## 技术实现说明

### 场景检测算法
使用帧差分直方图比较法：
1. 根据视频帧率自动确定采样率
2. 计算每帧的颜色直方图
3. 使用卡方距离（Chi-square）比较相邻帧的直方图差异
4. 差异超过阈值则判定为场景切换点
5. 支持流式检测，边读边分析，实时反馈进度
- **灵敏度**：可配置阈值（0.1~0.9）
- **最短场景时长**：0.5秒，避免闪烁误检

### 缩略图生成
- 使用 AVAssetImageGenerator 提取视频帧
- **双层缓存**：
  - 内存缓存：LRU 策略，最多 500 张
  - 磁盘缓存：PNG 格式，上限 2GB，自动淘汰旧数据
- 每个片段根据时长自动生成 2~6 张缩略图，均匀分布在时间范围内
- 支持批量预加载，进入网格时预生成缩略图

### 时间码处理
- 支持多种帧率（24/25/30/60fps 等）
- 标准 HH:MM:SS:FF 格式输出
- 支持丢帧时间码（drop-frame，30fps 时使用 `;` 分隔符）
- EDL 导出使用标准 CMX 3600 时间码格式

### 导出格式
1. **EDL (CMX 3600)**：行业标准编辑决策列表格式，支持多源文件
2. **FCPXML 1.9**：Final Cut Pro XML格式，兼容DaVinci Resolve和Premiere Pro，XML 属性安全转义

## 项目结构

```
Framwise/
├── App/
│   └── FramwiseApp.swift          # App入口、全局状态、键盘快捷键
├── Models/
│   ├── VideoClip.swift            # 视频片段模型（时间码、缩略图时间点）
│   └── ImportSession.swift        # 导入会话管理（片段集合、自定义排序）
├── ViewModels/
│   ├── VideoImportViewModel.swift # 导入+切割逻辑（流式分析、灵敏度/目标数配置）
│   ├── ClipGridViewModel.swift    # 网格+选择逻辑（搜索、过滤、分组）
│   ├── ExportViewModel.swift      # EDL/FCPXML导出（智能命名）
│   └── PreviewViewModel.swift     # 片段预览播放控制
├── Views/
│   ├── ContentView.swift          # 主界面布局
│   ├── DropZoneView.swift         # 拖拽导入区域+进度显示
│   ├── ClipGridView.swift         # 网格视图+拖拽重排+时间线导航
│   ├── ClipCellView.swift         # 单格视图+动画缩略图
│   └── ClipPreviewView.swift      # 片段预览弹窗+播放器
├── Services/
│   ├── SceneDetector.swift        # 场景检测（直方图差分、流式分析）
│   └── ThumbnailGenerator.swift   # 缩略图生成+双层缓存
└── Utils/
    └── TimecodeUtils.swift        # 时间码格式化（多帧率、丢帧支持）
```
