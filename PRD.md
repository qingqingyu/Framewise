# Framwise 产品需求文档 v5

## 产品定位

一个 Mac 原生桌面 App（Swift + SwiftUI），面向画面感剪辑师，帮助他们快速浏览、筛选和整理本地视频素材，自动过滤废料、识别重复镜位，然后导出到 PR/达芬奇进行精剪。

## 核心工作流

```
导入素材（文件/文件夹）
  → 自动切割成镜头网格
  → 废料检测 + 相似镜位分组
  → 浏览筛选（标签、手动纠正）
  → 导出 EDL/XML
  → 进 PR/达芬奇精剪
```

## 功能需求

### 1. 素材导入
- 支持拖拽或选择本地视频文件导入（支持多文件同时导入）
- 支持拖入文件夹，递归扫描其中的视频文件（上限 5000 个，符号链接循环安全）
- 素材文件始终保留在用户本地，不上传服务器
- 支持格式：MP4、MOV、MXF、AVI、MKV、M4V
- 导入过程实时显示进度（导入进度 + 分析进度）
- 导入完成后自动启动场景分析 → 废料检测 → 相似检测，实时显示分析状态

### 2. 自动切割逻辑
- **第一层**：按镜头 cut 点自动切割，每个自然镜头为一格
- **第二层**：根据目标片段数计算理想时长，超过理想时长的长镜头按比例再切割
- 目的是保证网格里每一格时长大致均匀
- **切片数量可调**（12~120，步进 12，默认 36）：用户在设置中通过单一滑块控制
- 场景检测灵敏度由系统根据切片数量自动推导（更多切片 = 更高灵敏度），用户无需手动调节
- 最短场景时长 0.5 秒，避免闪烁误检

### 3. 废料自动检测
- 导入分析完成后自动对每个片段进行废料检测
- 检测四种废料类型：
  - **Blackout**（黑屏）
  - **Dark**（极暗）
  - **Solid**（纯色/地面）
  - **Blurry**（模糊/失焦，基于拉普拉斯方差）
- 每个片段在 25%、50%、75% 时间点采样，投票决定废料类型
- 废料片段在网格中标记红色标签，可一键隐藏
- 导出时自动排除废料片段
- **手动纠正**：右键菜单可将废料标记为非废料（"KEPT"标记），或将正常片段标记为废料，支持批量操作
- 纠正结果通过 `effectiveWasteType` 覆盖自动检测结果

### 4. 相似/冗余片段检测
- 对非废料片段进行感知哈希（pHash）计算
- 相似度高于阈值（Hamming 距离 ≤ 10）的片段自动分组
- 每组选择时长最长的片段作为代表
- 网格中显示 "N TAKES" 标记，标识同一镜位的不同 take
- 支持按相似组过滤，快速在同组片段间导航
- 帮助用户从每组中选最好的一条，素材量可压缩到 1/3~1/5

### 5. 网格浏览界面
- 所有镜头以缩略图形式展示在网格中
- 每个片段根据时长生成 1~5 张缩略图并自动轮播
- 只渲染用户视野内的格子（懒加载），保证流畅度
- **网格大小可调**：小 / 中 / 大 三档，列数自适应窗口宽度
- **按源文件分组**：多文件导入时，网格按源文件自动分组显示，每组显示文件名和片段数量
- **搜索功能**：按文件名搜索片段
- **按源文件过滤**：点击侧边栏可按源文件筛选网格中的片段
- **视图模式切换**：可切换查看「全部片段」或「已选片段」
- **排序方式**：支持按原始顺序 / 按时长 / 按文件名 / 按相似组排序
- **隐藏废料**：一键隐藏废料片段，聚焦有用内容
- **相似分组**：可按相似组聚合显示，或按特定相似组过滤
- **迷你时间线导航**：网格顶部显示时间线条，按源文件着色，可点击快速定位到对应片段
- **键盘快捷键**：
  - `Space` 预览当前焦点或悬停片段
  - `方向键` 在网格中移动键盘焦点，自动滚动到焦点位置
  - `Enter` 预览当前焦点片段
  - `Escape` 清除键盘焦点
  - `1`~`9` 给焦点/选中/悬停片段快速切换对应位置的标签（toggle）
  - `Cmd+I` 打开导入
  - `Cmd+E` 打开导出
- **键盘焦点**：独立于鼠标选中态和悬停态，暖色虚线框标识；鼠标点击会同步焦点位置

### 6. 片段拖拽重排
- 用户可拖拽片段自由调整排列顺序
- **首次拖拽后切换为扁平网格**：从默认的按源文件分组视图，切换为单层扁平网格，用户自定义排序
- 拖拽视觉反馈：拖拽中源片段半透明，目标片段蓝色边框高亮
- **恢复原始排序**：工具栏提供「恢复原始排序」按钮，清除自定义顺序回到按源文件分组视图

### 7. 镜头选择
- 单击选中/取消某一格镜头
- 支持多选
- 选中的镜头高亮显示
- **批量选择操作**：
  - 全选
  - 取消全选
  - 反选（基于当前显示的片段）
  - 按同文件全选（右键菜单）
- **右键菜单**：选中/取消、按同文件全选、查看相似 take、预览、废料纠正、标签管理

### 8. 标签系统
- 支持创建自定义标签（名称 + 颜色）
- 可为片段分配/移除标签，支持批量操作
- 侧边栏显示标签列表，点击按标签过滤，每个标签旁显示快捷键编号
- 内置婚礼剪辑预设标签集（新娘准备、新郎准备、仪式、晚宴、第一支舞、花絮）
- **新建 session 时自动加载婚礼预设**
- **标签空状态引导**：标签列表为空时显示引导界面，引导用户加载预设或创建自定义标签
- **快捷键提示**：标签加载后，侧边栏底部显示 "Press 1–N to tag" 提示
- 右键菜单可快速分配标签或按标签全选

### 9. 片段预览
- 点击 Space 键或右键菜单可预览片段
- 预览弹窗内嵌视频播放器，自动播放该片段对应的时间范围
- **默认 2x 播放速度**，支持 1.0x / 1.5x / 2.0x / 3.0x / 4.0x 循环切换
- 播放控制：播放/暂停、进度条拖拽定位、回到起点
- 显示片段文件名和时间码范围（IN / OUT / DURATION）

### 10. 导出
- 用户选好镜头后，导出 EDL 或 FCPXML 文件
- 文件内记录原始素材文件名、每段片段的入点和出点时间码
- 导出的文件可直接导入 Premiere Pro 和 DaVinci Resolve
- **智能命名**：当所有选中片段来自同一个源文件时，自动以源文件名作为导出文件名；多源文件时使用时间戳命名
- 废料片段自动排除，不参与导出

### 11. 设置窗口
- 入口：菜单栏 `Framwise > Settings...` 或 `Cmd + ,`
- **Preview Tiles 滑块**（12~120，步进 12，默认 36）：控制每个视频生成多少个预览切片
- 滑块下方显示密度提示：Broad overview / Balanced / Detailed / Fine detail
- 场景检测灵敏度根据切片数量自动推导，用户无需关心
- 设置变更在下次导入时生效

### 12. Session 持久化
- 分析结果自动保存到本地 JSON 文件（`~/Library/Application Support/Framwise/session.json`）
- 包含所有片段数据、选择状态、自定义排序、标签、废料纠正、相似分组
- 应用重启后自动恢复上次会话
- 恢复时校验源文件是否仍然存在（通过文件大小 + 修改时间），自动清理失效数据
- 防抖保存（0.5s），避免频繁写盘

## 技术要求

- **平台**：macOS 原生，Swift + SwiftUI
- **视频处理**：AVFoundation 框架（帧提取、缩略图生成）
- **图像处理**：Core Image（帧分析、废料检测、pHash 计算）
- **无需服务器**，纯本地运行

## 不做的事情（明确边界）

- 不做时间轴剪辑功能
- 不做云端存储
- 不做自动成片
- 不做口播/字幕相关功能
- 不做视频转码
- 不做多语言本地化（当前仅英文界面）

## 技术实现说明

### 场景检测算法
使用帧差分直方图比较法：
1. 根据视频帧率自动确定采样率
2. 计算每帧的 RGB 颜色直方图（64 bins）
3. 使用卡方距离（Chi-square）比较相邻帧的直方图差异
4. 差异超过阈值则判定为场景切换点
5. 支持流式检测，边读边分析，实时反馈进度
- **灵敏度自动推导**：根据用户设定的切片数量线性映射（12 片 → 0.3，120 片 → 0.9），内部阈值为 `1.0 - sensitivity`
- **最短场景时长**：0.5 秒，避免闪烁误检

### 废料检测算法
对每个片段在 25%、50%、75% 时间点采样帧：
- **Blackout**：平均亮度极低
- **Dark**：平均亮度低于暗场阈值
- **Solid**：像素颜色方差极小（纯色画面）
- **Blurry**：拉普拉斯方差低于清晰度阈值
- 三帧投票机制确定最终废料类型

### 相似检测算法
使用感知哈希（pHash）+ Union-Find 分组：
1. 将帧缩放到 32×32 灰度图
2. 计算 2D DCT，取低频 8×8 区域
3. 以中位数为阈值生成 64-bit 哈希
4. Hamming 距离 ≤ 10 判定为相似
5. Union-Find 合并相似片段为一组
6. 每组选最长片段为代表

### 缩略图生成
- 使用 AVAssetImageGenerator 提取视频帧
- **双层缓存**：
  - 内存缓存：LRU 策略，最多 500 张
  - 磁盘缓存：PNG 格式，上限 2GB，自动淘汰旧数据
- 每个片段根据时长自动生成 1~5 张缩略图，均匀分布在时间范围内
- 支持批量预加载，进入网格时预生成缩略图

### 时间码处理
- 支持多种帧率（24/25/30/60fps 等）
- 标准 HH:MM:SS:FF 格式输出
- 支持丢帧时间码（drop-frame，30fps 时使用 `;` 分隔符）
- EDL 导出使用标准 CMX 3600 时间码格式

### 导出格式
1. **EDL (CMX 3600)**：行业标准编辑决策列表格式，支持多源文件
2. **FCPXML 1.9**：Final Cut Pro XML 格式，兼容 DaVinci Resolve 和 Premiere Pro，XML 属性安全转义
